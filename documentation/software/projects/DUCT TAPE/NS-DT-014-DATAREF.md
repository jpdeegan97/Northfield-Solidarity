## Required document sections for a Build Bundle
Every delivered bundle must include:
- Summary (what you said, in clean form)
- Problem statement
- Target user / audience
- Goals and constraints
- MVP scope
- Decisions + tradeoffs
- Roadmap (30/60/90)
- Risks + mitigations
- Next-step options (keep / iterate / build / IP posture)

## File formats
- Default: Markdown + PDF export
- Optional: Docx export (for client editing)

## Metadata footer (every artifact)
- project: Duct Tape Sessions
- session_id
- bundle_id
- bundle_version
- generated_at (UTC)
- confidentiality label (Private / Shareable / Public)

## Retention policy (default)
- Raw media: 30 days (configurable per contract)
- Transcript + bundle: 180 days (configurable)
- Audit logs: 1 year minimum
- Deletion on request: supported (subject to contract / legal holds)

## Consent records
- Store signed consent artifacts linked to session_id
- Record attendee list and consent status
- Without complete consent, publication is blocked

## Template references
- Intake form template
- Consent form template (speaker + attendee)
- Session facilitation checklist
- QA/redaction checklist
- IP posture one-pager (ownership/license/buyout)
