## Session lifecycle
1) **Intake**
   - objective captured
   - boundaries and consent confirmed
2) **Schedule & payment**
3) **Pre-brief**
   - confirm “no interruptions” mechanic
   - confirm what “success” looks like for the speaker
4) **Capture**
   - setup (5 minutes)
   - uninterrupted talk (30 minutes)
   - optional clarifying questions (up to 10 minutes)
5) **Ingest & secure storage**
6) **Processing**
   - transcript → structure → synthesize → QA → package
7) **Delivery**
   - portal link + bundle version v1.0
8) **Iteration (optional)**
   - daily improvements: v1.1, v1.2...
9) **Closeout**
   - retention schedule starts
   - IP posture finalized (ownership/license/buyout)
   - deletion if requested

## Artifact lifecycle
- Draft → QA’d → Delivered → Revised → Deprecated → Archived/Deleted
- Every delivery is versioned and reproducible from pipeline inputs (as feasible)

## Engagement lifecycle
- One-off session
- Session + daily iteration subscription
- Session → IP buyout
- Session → build engagement (SOW)
