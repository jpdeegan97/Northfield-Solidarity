## Core terms
**Session**: the live capture event (setup + talk + debrief).  
**Speaker**: the person delivering the uninterrupted monologue.  
**Audience**: optional trusted attendees (friends/family).  
**Facilitator**: ensures consent, boundaries, timing, and flow.  
**Capture Kit**: camera(s), mic(s), lighting, storage, sync clock.  
**Processing Pipeline**: ingestion → transcription → structuring → QA → publish.  
**Build Bundle**: the delivered artifact pack (docs + roadmap + options).  
**Iteration Day**: paid day where enhancements/optimizations are produced.  
**IP Package**: assignments/licenses, provenance notes, and deliverable chain.

## Session types
- **Founder Burst** (default): product idea → MVP plan
- **Ops Unknot**: messy process → workflow + system design
- **Pitch Forge**: story → deck narrative + proof points
- **Spec Sprint**: concept → requirements + APIs + data model
- **Roadmap Rescue**: scattered tasks → prioritized plan

## Artifact classes
- **Narrative artifacts**: vision, problem, customer, value props
- **Specification artifacts**: requirements, APIs, data model
- **Execution artifacts**: roadmap, backlog, milestones, risk register
- **Governance artifacts**: IP posture, consent logs, retention policy

## Capability tiers
**Tier 0 (Instant)**: auto-processed bundle, minimal QA  
**Tier 1 (MVP)**: auto + human QA pass, improved clarity  
**Tier 2 (Pro)**: deeper synthesis, comparisons, market scan (optional)  
**Tier 3 (Build)**: implementation engagement (contracted)

## Conduct & boundaries
- Uninterrupted speaking is enforced **without intimidation**.
- No harassment, coercion, or sexualized conduct.
- Attendees must agree to ground rules and may be removed for violations.
