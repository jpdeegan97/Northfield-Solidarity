# NS-TINE-004-LIFECYCLE — Lifecycle

## User Journey Lifecycle
**Onboard → Intent → Menu → Procurement → (Optional) Chef → Feedback → Repeat**

### Onboard
- Preferences, allergies, diet, budget, equipment, pantry staples
- Location + store selection preferences

### Intent Session
- User provides a request (duration, healthiness, cuisines, schedule)

### Plan
- Menu generated and reviewed/edited
- Nutrition and allergen risk review

### Procure
- Cart options generated
- Substitutions confirmed
- Checkout executed (partner or store)

### Fulfill (optional)
- Chef match → booking → prep → cook → handoff → rating

### Feedback Loop
- Taste feedback, substitutions accepted/rejected, delivery performance
- Updates personalization model and “do-not-suggest” lists

## Artifact Lifecycle
- Draft menu → user-approved menu → cart-ready → ordered → completed
- Chef candidate → verified → active → booked → completed → reviewed → retained/suspended
