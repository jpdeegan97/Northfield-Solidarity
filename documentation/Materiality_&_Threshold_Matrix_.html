Purpose

This document defines quantitative and qualitative thresholds that trigger approvals, escalations, overrides, or contingency actions across all Northfield Solidarity SOPs.

It exists to:

  • Convert policy into executable rules
  • Prevent ad‑hoc decision making
  • Enable automation in proprietary systems
  • Ensure proportional governance (no over‑ or under‑control)

This matrix is binding across all SOPs unless explicitly overridden by Parent / HoldCo.

Governing Principles

  • Materiality is contextual, not absolute
  • Higher risk = lower tolerance
  • Liquidity, compliance, and IP are always high‑sensitivity domains
  • Thresholds are reviewed at least annually

Override Authority

  • Parent / HoldCo may override thresholds in writing
  • Overrides must be:
  • Time‑bound
  • Documented
  • Reviewed post‑event

System Implementation Notes

This matrix should map to:

  • Rule engines
  • Approval workflows
  • Alert triggers
  • Incident state machines

Thresholds should be configurable, not hard‑coded.

Review & Updates

  • Reviewed annually
  • Updated upon:
  • Material scale change
  • Regulatory change
  • Incident learnings
