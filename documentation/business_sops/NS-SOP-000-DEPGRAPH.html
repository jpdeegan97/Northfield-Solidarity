Purpose

This document provides a single authoritative map of all Standard Operating Procedures (SOPs) within the Northfield Solidarity governance framework.

It exists to:

  • Show how SOPs relate, defer, and escalate
  • Eliminate overlap and contradiction
  • Clarify authority, precedence, and failure modes
  • Serve as the backbone for system implementation

Implementation Guidance (System Design)

Each SOP should map to:

  • A module in your proprietary system
  • Explicit permissions and approval gates
  • Logged events and overrides

Recommended core tables / objects:

  • Entities
  • SOP Authority Levels
  • Approval Thresholds
  • Incident States
  • Intercompany Agreements
  • IP Assets

Change Management

  • SOP changes must not violate hierarchy
  • Lower-level SOPs may not override higher-level SOPs
  • Material changes require Parent approval
